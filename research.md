# 연구 보고서: 무료 이미지 호스팅 및 티스토리 연동 전략

## 1. 개요
사용자는 생성된 이미지를 무료 호스팅 서비스에 업로드하고, 해당 외부 URL을 포함한 HTML 코드를 티스토리에 직접 붙여넣어 완벽하게 작동하도록 하는 방안을 원함. 과거 '옵시디언 이미지 처리용' 계정을 사용한 경험이 있음.

## 2. 후보 무료 호스팅 서비스 분석

### 2.1. Cloudflare Pages / R2 (추천)
- **근거**: 사용자 환경에서 이미 Cloudflare('dokyungja' 계정)를 사용 중이며 wrangler 및 deployment 스크립트가 존재함.
- **장점**: CDN 성능이 매우 우수하며, GitHub 연동 시 자동 배포가 가능함. 영구적인 URL 제공.
- **단점**: R2는 초기 설정이 약간 필요함.

### 2.2. GitHub (GitHub Pages / Raw Content)
- **근거**: 많은 옵시디언 사용자들이 'Image Auto Upload' 플러그인을 사용하여 GitHub 저장소에 이미지를 업로드함.
- **장점**: 관리가 쉽고 무료이며, 티스토리 등 외부 서비스에서 접근 시 안정적임.
- **단점**: 간혹 이미지 URL이 `raw.githubusercontent.com`인 경우 로딩 속도가 CDN보다는 느릴 수 있음.

### 2.3. Cloudinary / ImgBB
- **근거**: 옵시디언 사용자들이 흔히 사용하는 이미지 전용 호스팅.
- **장점**: 이미지 크기 조절, 최적화 등 부가 기능 제공.
- **단점**: 무료 할당량(대역폭) 제한이 있을 수 있음.

### 2.4. Supabase Storage
- **근거**: `<persistent_context>`에 'Supabase Keep Alive' 액션 주기가 언급되어 있어 이미 사용 중일 확률이 높음.
- **장점**: 이미 배포된 인프라를 활용 가능.

## 3. 과거 옵시디언 이미지 처리 방식 추정
옵시디언 사용자라면 다음 중 하나일 가능성이 큼:
1.  **GitHub 리포지토리 + jsDelivr**: 가장 보편적인 방식.
2.  **Cloudinary**: 이미지 최적화까지 한꺼번에 처리할 때 사용.
3.  **Imgur**: 간단한 업로드 시 사용.

## 4. 제안하는 워크플로우
1.  기존에 사용 중인 **Cloudflare** 또는 **GitHub** 계정을 활용하여 이미지를 호스팅함.
2.  현재 로컬의 `images/` 폴더 내 이미지를 해당 저장소로 업로드.
3.  이미지 주소를 외부 URL(https://...)로 치환.
4.  최종 HTML을 복사하여 티스토리에 'HTML 모드'로 붙여넣기.

## 5. 결론 및 다음 단계
사용자가 Cloudflare나 GitHub 중 선호하는 방식을 선택하면, 해당 방식으로 자동 업로드 및 URL 변환을 수행하는 `plan.md`를 작성할 예정.
