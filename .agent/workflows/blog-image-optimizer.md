# 🚀 티스토리 이미지 통합 배포 워크플로우

이 워크플로우는 블로그 포스트용 HTML 내의 이미지 주석을 감지하여 실시간으로 이미지를 생성하고, 이를 GitHub 저장소에 업로드한 뒤, CDN 주소(jsDelivr)와 SEO 태그가 포함된 형태로 HTML을 최종 업데이트합니다.

## 📋 사전 준비 사항
1.  **HTML 파일**: 내부에 `<!-- [이미지 명칭: 상세 설명] -->` 형태의 주석이 포함되어 있어야 합니다.
2.  **GitHub 저장소**: `every-git/tistory-assets`와 같은 이미지 호스팅용 공개 저장소가 연결되어 있어야 합니다.

## 🛠️ 자동화 프로세스 단계

1.  **이미지 명세 추출**: HTML 파일 내의 모든 이미지 주석을 찾아 프롬프트와 SEO 파일명을 설계합니다.
2.  **이미지 생성 (`generate_image`)**: 'Simple 3D Icon' 스타일로 고해상도 이미지를 생성합니다.
3.  **SEO 최적화 리네임**: 각 이미지의 파일명을 핵심 키워드가 포함된 영문 파일명으로 변경합니다.
4.  **GitHub 동기화 (`git push`)**: `images/` 폴더의 변경 사항을 원격 저장소에 즉시 업로드합니다.
5.  **HTML CDN 인젝션**:
    *   로컬 경로를 `https://cdn.jsdelivr.net/gh/...` 형태의 외부 URL로 교체합니다.
    *   `alt`, `title`, `figcaption` 등의 SEO 및 접근성 태그를 자동 삽입합니다.

## 💬 실행을 위한 명령 예시 (앞으로 이렇게 말씀해 주세요)

> **"이 [파일명].html 파일에 대해 이미지 자동 배포 프로세스 시작해줘."**
> **"포스트 내 주석을 바탕으로 이미지 생성하고 깃허브에 배포해서 HTML 완성해줘."**
